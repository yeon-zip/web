# Library App Master Doc

## 1. 제품 개요

### 목표
- 위치 기반으로 주변 도서관을 찾고, 원하는 책의 소장 여부와 대출 가능 상태를 한 번에 확인하는 웹 앱을 만든다.
- 검색, 상세, 저장, 투표 흐름을 목업 중심으로 빠르게 구현할 수 있도록 제품 규칙과 화면 경계를 먼저 고정한다.

### 현재 범위
- 이번 문서는 제품 명세, 화면 경계, 도메인 엔티티, 상태 규칙, 구현 우선순위를 정의한다.
- 실제 API 연동, 인증 방식, 배포 환경은 이후 단계에서 확정한다.

### 핵심 기능 묶음
- 통합 도서 검색: 제목 키워드 하나로 주변 도서관 소장 정보를 거리 기준으로 통합 조회
- 위치 기반 도서관 조회: GPS 또는 수동 위치 기준 반경 `5km`, `15km`, `30km` 탐색
- 대출 상태 조회: `대출 가능`, `대출 중`, `예약 가능` 상태와 가용성 필터
- 도서 상세 조회: 표지, 줄거리, 가격, 저자, 출판사, 보유 도서관 목록
- 도서관 상세 조회: 운영 시간, 정기 휴관일, 법정 공휴일, 임시 휴관일
- 도서 추천/비추천: 사용자별 1회 투표, 수정 가능, 집계 노출
- 도서 찜 / 도서관 찜: 저장, 해제, 목록 조회, 상세 이동

### 기본 제약
- 목업 우선: 화면 구조와 정보 우선순위는 제공된 6개 목업을 기준으로 유지한다.
- 기술 스택 우선순위는 `Next.js`, `React`, `TypeScript`, `React Query`, `CSS Modules`, `FSD` 구조를 따른다.

### 공통 화면 요소
- 상단 앱 쉘: 프로젝트 이름, 찜 진입, 로그인 버튼
- 공통 카드: 통계 카드, 도서 카드, 도서관 카드
- 공통 칩: 거리, 운영 상태, 대출 상태, 태그
- 공통 액션: 검색, 상세 보기, 찜 추가/해제, 추천/비추천

### 화면 경계 메모
- `S01`과 `S06`은 같은 검색 플로우로 묶는다.
- `S04`와 `S05`는 도서 상세에서 이어지는 상세 플로우로 묶는다.
- `S02`와 `S03`은 저장 플로우로 묶되, 데이터 모델은 책 찜과 도서관 찜으로 분리한다.

## 4. 상태/필터 규칙

### 위치 및 반경
- 기본 반경 옵션은 `5km`, `15km`, `30km` 세 개만 허용한다.
- 위치 소스는 GPS와 수동 위치 설정 두 가지다.
- 같은 검색 세션 안에서는 마지막 반경과 위치 설정을 유지한다.

### 검색 및 정렬
- 도서 검색은 제목 키워드 기준으로 시작한다.
- 검색 결과의 기본 정렬은 거리 오름차순이다.
- 통합 검색 결과는 대표 도서 정보와 주변 소장 도서관 요약을 함께 보여준다.

### 대출 가능 필터
- `availabilityOnly = false` 면 해당 도서를 보유한 모든 도서관을 보여준다.
- `availabilityOnly = true` 면 현재 `대출 가능` 상태가 1권 이상 있는 도서관만 남긴다.
- 필터 토글 시 결과 리스트와 요약 수치가 함께 즉시 갱신된다.

### 도서 상세 및 투표
- 한 사용자는 한 도서에 대해 추천/비추천 중 하나만 가질 수 있다.
- 기존 투표가 있으면 같은 도서에서 선택을 수정할 수 있다.
- 비로그인 상태의 투표 액션은 로그인 유도 상태로 처리한다.

### 찜 규칙
- 도서 찜과 도서관 찜은 별도 컬렉션으로 저장한다.
- 같은 대상을 중복 찜하지 않는다.
- 찜 해제 시 해당 목록 카드가 즉시 제거되거나 빈 상태가 갱신된다.

### 도서관 운영/휴관 정보
- 도서관 상세에는 정기 휴관일, 법정 공휴일, 임시 휴관일을 모두 보여줄 수 있어야 한다.
- 실데이터 연동 전까지는 휴관 정보를 엔티티 계약 수준으로만 고정한다.

## 5. 구현 순서

### Phase 0. 문서와 운영 규칙 고정
- 본 문서와 프로젝트 운영 문서를 먼저 만든다.
- 새 세션에서도 제품 규칙과 작업 이력을 문서만으로 빠르게 복기할 수 있어야 한다.

### Phase 1. 앱 쉘과 공통 UI
- 상단 네비게이션, 레이아웃 컨테이너, 카드, 칩, 버튼, 모달, 통계 박스를 구현한다.
- 공통 스타일 토큰은 UI Foundation 워커가 먼저 소유한다.

### Phase 2. 검색 플로우
- `S01` 검색 홈
- `S06` 통합 검색 결과
- 거리 반경 선택, 거리순 정렬, 대출 가능 필터, 대표 검색 결과 요약

### Phase 3. 상세 플로우
- `S04` 도서 상세 모달
- `S05` 도서 보유 도서관 리스트
- 추천/비추천, 도서관 상태 표기, 휴관/운영 정보 노출

### Phase 4. 저장 플로우
- `S02` 저장한 도서관 목록
- `S03` 저장한 도서 목록
- 로그인 경계, 찜 해제, 저장 목록 빈 상태

### Phase 5. 계약 점검과 회귀 확인
- 빈 상태, 오류 상태, 비로그인 상태, 필터 조합 상태 점검
- 추후 실데이터 단계에서는 이 phase를 API 통합 phase로 전환한다.

### 독립 스모크 시나리오
- 검색: `S01`에서 키워드 입력 후 `S06`으로 이동하고 대표 도서와 도서관 요약이 보여야 한다.
- 거리 필터: `5km`, `15km`, `30km` 변경 시 결과 수와 리스트가 갱신되어야 한다.
- 대출 가능 필터: 토글 시 `대출 가능` 도서관만 남아야 하며 해제 시 전체가 복원되어야 한다.
- 도서 상세/투표: `S06` 또는 `S03`에서 상세 진입 후 투표 수와 사용자 선택 상태가 맞아야 한다.
- 찜 해제: `S02`, `S03`에서 해제 시 카드와 카운트가 함께 갱신되어야 한다.
- 도서관 상세: `S05`에서 도서관 상세 진입 시 운영 상태와 휴관 정보가 일관되게 보여야 한다.
